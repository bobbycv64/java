Tomcat & MySQL Setup Instructions

This page describes the procedures to download, install and configure Tomcat (Java Server Page (JSP) Server) and MySQL (Relational Database) for a Web Enabled Database.
Downloads

MySQL Database is a world leader in Relational Databases and open source technology.  This is the database that we will be interacting with.  Connector J contains the Java DataBase Connectivity jar files which Tomcat needs to communicate to the MySQL server.  Tomcat is a Java Server Page (JSP) architecture, model and program for front end web page processing.  Download the various products from the following links below:
MySQL Database
 http://dev.mysql.com/downloads/mysql/
Connector J
 http://dev.mysql.com/downloads/connector/j/
Tomcat
 http://jakarta.apache.org/tomcat/
Tomcat JSTL
 http://cvs.apache.org/builds/jakarta-taglibs/nightly/projects/standard/

Installation

  • Install and configure the MySQL Database, detailed instructions are described here:  MySQL Installation and Configuration
  • Install and configure the Tomcat JSP Server, detailed instructions are described here:  Tomcat Installation and Configuration
  • Unzip the ConnectorJ Java ARchive (JAR) file and place in the %CATALINA_HOME%\lib directory, e.g.
copy mysql-connector-java-5.1.6-bin.jar c:\tomcat\lib\mysql-connector-java-5.1.6-bin.jar
  • Unzip the Tomcat JSTL JAR file and place in the %CATALINA_HOME%\lib directory, e.g.
copy jstl.jar c:\tomcat\lib\jstl.jar
copy standard.jar c:\tomcat\lib\standard.jar
MySQL Commands

The following are useful commands for MySQL Adminstrative purposes:

command
 mysql> \. externalSQLFile.sql
 Example of executing an external SQL file.
mysqladmin
 mysqladmin -u username -p command
 Logging into the mysql server and issueing a command.  The password will be prompted after you press carriage return.
mysqladmin
 mysqladmin -u root -p command
 Example of root logging in.
mysql
 mysql -u username -p
 Logging into the mysql command line editor.
mysql
 grant all privileges on *.* to root@localhost with grant option;
 Opens up all databases and tables in the MySQL Database.
telnet
 telnet localhost 3306
 Testing if MySQL is online.

MySQL Installation and Configuration

The following are the steps and instructions for installing the MySQL Server.


After downloading the MySQL software, unzip the zip file and execute the setup.exe program.  This will bring up the screen above.  Select Custom and then press Next >.

Press the Change... button and modify the installation directory to c:\mysql or a root level directory.  Press Next >.

Your selections will be presented.  Make sure that the directory is where you want and then press Install.

You will be asked for registration, select Skip Sign-up and then press Next >

The MySQL files will now be copied to your system.  We now configure MySQL for your system.  Press Finish.

Select Standard Configuration and press Next >.

Select Install As Windows Service and Include Bin Directory in Windows PATH and then press Next >.

Enter a root password with confirmation and check if you want root access from remote machines.  Press Next >.  MySQL will start up.  If you get a port 3306, default port for MySQL.  See the next screens.

Get to this screen by clicking on Start/Settings/Control Panel/Administrative Tools/Services and find the MySQL service. 

Press the Stop button in the Service status panel.  Then remove MySQL from the system all together before trying a newer instance.

Once everything is set, this screen appears.  Press Execute and MySQL will prepare configuration, write configuration, start service and apply security settings.  MySQL should be started.

Tomcat Basic Installation and Configuration

The following are the steps and instructions for installing the Tomcat Server.

Note:  %CATALINA% and TOMCAT_ROOT are synonymous.
  • Unzip the Tomcat.zip file as a main directory on the local system, this will create the CATALINA environment variable and value, e.g. c:\tomcat
  • Change directory to the %CATALINA%\bin directory and create two desktop shortcuts.  The first shortcut will execute the startup.bat file and the second shortcut will execute the shutdown.bat file.
  • Change directory to %CATALINA%\webapps and create a new directory for your JSP files, e.g. c:\tomcat\webapps\cv64, this will be where your JSP files will reside.
  • Change directory to your newly created webapps directory and create a new directory for your XML files, e.g. c:\tomcat\webapps\cv64\WEB-INF, this is where your XML files will reside.
  • Add the following JSP code to a file called Test.jsp  to the newly created JSP directory as shown in Figure Tomcat-1:
<!DOCTYPE html PUBLIC "-//w3c//dtd html 4.0 transitional//en">
<html>
  <head>
    <title>Test Page</title>
  </head>

  <body>
    <p>Test Page</p>
  </body>
</html>
Figure Tomcat-1 Test.jsp
  • Add the following XML code to a file called web.xml  to the newly created XML directory as shown in Figure Tomcat-2:
<?xml version="1.0" encoding="ISO-8859-1"?>
<web-app xmlns="http://java.sun.com/xml/ns/j2ee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-app_2_4.xsd"
    version="2.4">

    <description>Test Page</description>
    <display-name>Test Page</display-name>
</web-app>
Figure Tomcat-2 web.xml

Tomcat is ready to start up and execute.  Double click the Start Tomcat shortcut and go to your webpage, e.g. http://localhost:8080/cv64/test.jsp, you should see the following:


Figure Tomcat-Test Page

Tomcat and MySQL Configuration

See Apache Instructions to configure Tomcat and MySQL connectivity:  http://jakarta.apache.org/tomcat/tomcat-5.0-doc/jndi-datasource-examples-howto.html

Note:  The Apache Instructions state to place the server.xml configurations between <Context> and </Host>.  There is no <Context> element so place just above the </Host> element.

Below is an example of changing the database values in the file %CATALINA%\conf\server.xml
<!-- Add MySQL ****************************************************************** -->

    <Context path="/cv64" docBase="cv64" debug="5" reloadable="true" crossContext="true">

        <Resource name="jdbc/cv64DB" auth="Container" type="javax.sql.DataSource"
                       maxActive="100" maxIdle="30" maxWait="10000"
                       username="root" password="bobby" driverClassName="com.mysql.jdbc.Driver"
                       url="jdbc:mysql://localhost:3306/cv64?autoReconnect=true"/>
    </Context>

<!-- Add MySQL ****************************************************************** -->
Figure %CATALINA%\conf\server.xml

Below is an example of the %CATALINA_HOME%\webapps\cv64\WEB-INF\web.xml file.

<?xml version="1.0" encoding="ISO-8859-1"?>
<web-app xmlns="http://java.sun.com/xml/ns/j2ee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee http://java.sun.com/xml/ns/j2ee/web-app_2_4.xsd"
    version="2.4">

    <description>Test Database</description>
    <display-name>Test Database</display-name>

    <resource-ref>
          <description>cv64 DB Connection</description>
          <res-ref-name>jdbc/cv64DB</res-ref-name>
          <res-type>javax.sql.DataSource</res-type>
            <res-auth>Container</res-auth>
    </resource-ref>
</web-app>
Figure %CATALINA_HOME%\webapps\cv64\WEB-INF\web.xml

Below is an example of the %CATALINA_HOME%\webapps\cv64\Test.jsp file.

<%@ taglib uri="http://java.sun.com/jsp/jstl/sql" prefix="sql" %>
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>

<sql:query var="rs" dataSource="jdbc/cv64DB">
select * from names
</sql:query>

<html>
  <head>
    <title>Names Database</title>
  </head>
  <body>

  <h2>Results</h2>
  
<c:forEach var="row" items="${rs.rows}">
    First ${row.first}<br/>
    Last  ${row.last}<br/>
</c:forEach>

  </body>
</html>
Figure %CATALINA_HOME%\webapps\cv64\Test.jsp

Restart Tomcat and go to your database webpage, e.g. http://localhost:8080/cv64/Test.jsp, you should see the following:
