Structured Programming

Information Technology (IT) and Object-Oriented (OO) Programming didn't exist or were not known as the popular practice of Structured Programming (SP).  Robert Estey began his computing career in 1985 and his first company invested almost $1 Million (United States) Dollars for a Digital Equipment (DEC) Virtual Address eXtension (VAX) 11/780 state-of-the-art computer with 1 MHz processor, 4 MB of memory, 120 MB disk drive and a magnetic tape drive.  The computer paradigm at the time was centralized computing with direct RS-232 terminal connections.  The popular programming languages were C, COBOL, Fortran and Pascal, written to solve simple problems.

While the computing industry was working with Structured Programming and centralization, the same issues from the past occur today. How do you correctly analyze, design and produce error free code to the customer? The answer then, "Hire more programmers."

Object-Oriented Programming

The first Object-Oriented Programming Language (OOPL), Simula was introduced in 1962, decades before the popular OOPLs Smalltalk 1980 and C++ in the mid 1980's.  Robert Estey with over a decade of Structured Programming  and many SP developers were in for a paradigm shift to learn the new architecture and terminology of OO.

Paradigm Shift

Today the world has a lot of structured code which is harder to maintain and impossible to develop in a timely manner. Object-Oriented Programming enables software developers to program dynamically, distributed, secure, multi-threaded, robust and high performance applications.

The World Wide Web (WWW) was one of the first true movements using Object-Oriented technology.  Web Browsers no matter who wrote them could interpret and display the contents on a world wide stage.  The WWW was created back in the 1960's but didn't get public awareness until the early 1990's.  In 1995, the Java Programming Language emerged and was accepted as one of the popular Web Programming Languages.

Java

The Java programming language is simple to program and has several built in features, e.g. Security, Threads, Object-Oriented design, the most important feature was TRUE Portability.  The Java Virtual Machine (JVM) executed on any platform guarantees portability, without recompiling, regenerating objects or any other inconvience of operating systems. Java is TRULEY PORTABLE.

C and C++ (Hybrid Object-Oriented C) were popular programming languages before the birth of the the WWW and have many of the same features that Java does, however, they didn't have TRUE PORTABILITY.  A C/C++ program didn't necessarily mean that the same program would work on all platforms.  C/C++ code required rewriting the code, using pre-processor commands or make file modifications.  The C community realized that there were serious porting issues that the Portable Operating System Interface eXchange (POSIX) standard was created.  The authors believe that this was to little, to late and C/C++ missed the WWW by not following a single standard strategy.

About This Book

Chapter 1 is an overview about Object-Oriented Concepts and the importance before programming Java.  Every educational system always jumps right into Java, which is a shame when the student needed OO fundamentals.

Chapter 2 introduces the History of Java, the eleven design goals and advantages/disadvantages of Java computing.  Why the computing industry, commercial, government, educational and private sectors embrased the technology and are moving to the Java programming language.  Chapter 2 also discusses the basics of the Java Virtual Machine and the Java Programming Environment.  This chapter also gives step by step instructions on how to install the Java Virtual Machine and write your first Java program, Helloworld.java.

Chapter 3 covers basic Class anatomy, attributes and methods. Fundamental basics and flow control are also discussed, e.g. for, do, while.
