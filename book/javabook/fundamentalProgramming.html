3.1. Documentation

3.1.1. Comments

Comments are used in source code to provide a reader with information on what the developer was thinking when writing the program. Comments are very useful because the originator is not available or to remind whomever what the code is doing. Comments in Java are noted by either // (for a single line) or /* and */ (for several lines).

The win-win situation for developers is no matter how much extra comments you put in a program, none of the comments will show up in the executable. If you don't believe me just try. Write a program with no comments. Compile and note the sizes of both the source and bytecode. Now edit the same program and add several comments. Compile and the source code will be larger as expected. However the bytecode is exactly the same size as the uncommented code. Below is an example of our first program with comments.

Figure 3.1. Comments Source Example

figures/fundamentals/HelloworldComments.java
          
/*  This is a multilined comment at the beggining of the program
 *  You can put as much text as you wish as long as (slash asterisk)
 *  and (asterisk slash) and are defined.  Remember you can also put
 *  (slash slash) on each line.
 */

public class HelloworldComments  { // Here’s a comment at the end of a line

  // This is the start of the main method.

  public static void main(String[] args) {

    // This line prints out Helloworld with Comments
    System.out.println("Helloworld with Comments");
  }

} // comments can go anywhere

3.1.2. javadoc (Java Documentation)

Java has included by default a documentation tool called javadoc, which means Java Documentation. javadoc reads documentation tags that contain the "@" (at sign) in a Java program and produces HyperText Modeling Language (HTML) web pages. javadoc is easy to write and helpful to anyone who wants to understand your programs. Below is an example of our first program Helloworld.java with some javadoc included.

Figure 3.2. Helloworld Javadoc

figures/fundamentals/HelloworldJavadoc.java

/** This is a multilined javadoc comment at the beginning
 *  of the program.  You can put as much text as you wish
 *  as long as the slash and double asterisk are at the 
 *  beginning and asterisk and slash are defined.
 *  Remember both javadoc and comments are allowed.
 */

/** Here is another javadoc comment on one line. */

/** 
 *  Here is a javadoc comment for classes.  The
 *  Helloworld class is a simple example of a Java
 *  program.
 *
 *  @version  2001.09.05
 *  @author  Robert Estey
 *  @author  Eric Richardson
 */
 public class HelloworldJavadoc { // Here’s a standard comment

  /**
   *  This is the main method starting the Java program.
   *
   *  @param args are the command line arguments sent
   *  into the main program.
   */ 
   public static void main(String[] args) { // comment

     // This line prints out Helloworld
     System.out.println("Helloworld with javadoc"); 
   } 

} // comments can go anywhere

The javadoc execution below will generate the author, version number, private attributes and methods of the Helloworld.java file and send the javadoc output to the -d directory. Look at the destination directory and look at all the work that javadoc produced with very little work on your part.

set JAVA_DOC_HTML=c:\myDirectory\javadoc

javadoc -author -version -private -d %JAVA_DOC_HTML% HelloworldJavadoc.java

Java as described before is explicit towards data types. An integer is 4 bytes, not 2 or 8. No matter what Java Virtual Machine (JVM) integer is always 4 bytes. 3.5. Statements

Java has a concept much like many other programming languages called a statement. A statement is a section of code that requires a computer to perform some work. A statement ends with the semicolon character (; \u003b). When entering code your statement is not complete without the semicolon. An example would be int i; This is stating that an integer attribute i is being declared.

3.6. Declarations and Assignments

Now that you have a idea about Naming and Statements you can start declaring and assigning attributes. An attribute as we recall is the state of an object. When declaring attributes they must have a type (object or primitive) declaration. This means the attribute when first being declared must have a type in front of the attribute name, e.g. int i; This is called declaration.

Newly created attributes that were declared should have an initialized state. Although Java predefines attributes according to their type, a new attribute should be set to an initialized state as a good programming practice. To initialize state the attribute receives an assignment. The assignment character is the equals character (= \u003d). Assignments are executed from right to left.

Figure 3.6. Declaration Assignment Example

figures/fundamentals/DeclarationAssignment.java

public class DeclarationAssignment {

  }
}

3.7.2. Increment / Decrement Operators

Java uses the same increment and decrement operators as C and C++. To increment any attribute use the plus plus characters, e.g. (++ \u00\u00). To decrement use the minus minus characters, e.g. (-- \u00\u00). Both of these operations add (increment) or subtract (decrement) the current value in an attribute by one.

The increment/decrement operators can be placed before or after the attribute. By placing the increment/decrement operator in front means to increment the attribute first then process any block of code after. Placing the increment/decrement operator after the variable means to increment the attribute after processing a block of code.

Figure 3.9. Increment Decrement Operators

figures/fundamentals/IncrementDecrementOperators.java
          
public class IncrementDecrement {

  public static void main(String[] args) {

    int i = 0;

    System.out.println(i);	// i = 0
    System.out.println(i++);	// i = 0
    System.out.println(i++);  // i = 1
    System.out.println(++i);  // i = 3
    System.out.println(i--);    // i = 3
    System.out.println(--i);	// i = 1
  }
}

3.7.4. Precedence

Java's many different symbols representing operations follow a strict rule of precedence. This means if you have a statement, e.g. 4 + 5 * 6; what order of computation occurs. Do you get the value 34 or 54? Precedence evaluates a statement in a given order by the type of symbols used. In the case above the statement would evaluate to 34 because multiplication is higher in the order of precedence.

The highest order is the parenthesis symbols (() \u00\u00) and good programming practice uses parenthesis to not only clarify work but for an operation that you wish. If you wanted to the statement 4 + 5 * 6; to compute using the addition first then put parenthesis around the attributes, e.g. (4 + 5) * 6. This would give the result of 54. 3.9. Formatting Output

To view output for your program the java.lang.System class contains methods that produce generic output features. To print the contents of a variable is as simple as System.out.println("message"); The word message appears in your output device, usually the screen. But there are times when formatting output is either required or desirable.

Java has the java.text.Format class that enables formatting locale-sensitive (specification for country specific properties) information such as dates, messages and numbers. The System class is used to print out the formatted information. The following diagram gives some examples of formatted output.

Figure 3.13. Formatting Output

figures/fundamentals/FormattingOutput.java
          
import java.text.DecimalFormat;

public class FormattingOutput {

  public static void main(String[] args) {

    DecimalFormat decimalFormat = new DecimalFormat("$,##0.##");
    float numerator   = 64.0f;
    float denomenator = 13.0f;
    int intValue = (int)numerator / (int)denomenator;	// 4
    float floatValue = numerator / denomenator;        	// 4.923077

    System.out.println("Integer Value: " + intValue);	// 4
    System.out.println("Float Value:   " + floatValue);	// 4.923077
    System.out.println("Small Dollars: " + 
      decimalFormat.format(floatValue));		// $4.92

    System.out.println("Large Dollars: " +
      decimalFormat.format(floatValue * 1000));	// $4,923.08
  }
}

3.10. Flow Control

A block of code is defined as one or more statements within a set of brackets ({} \u00\u00). Java just like most programming languages can control how we process information through the concept of Flow Control. Java has three basic flow controls conditional, looping and selection.

3.10.1. Conditional Flow Control

Conditional flow control is performed by the if-else commands. If a boolean (logical) condition is true then do the block of code else do the else block of code. If there is no else block then do nothing.

Figure 3.14. Conditional Flow Control

figures/fundamentals/ConditionalFlowControl.java
          
public class ConditionalFlowControl { 

  public static void main(String[] args) {

    int smallInt = 13;
    int largeInt = 64;

    //  == is a logical comparison, not assignment.
    if (smallInt == largeInt) {
        System.out.println("This will never execute");
    } else {
        System.out.println(smallInt + " isn't equal to " + largeInt);
    }

    if (smallInt < largeInt) {
      System.out.println(smallInt + " is smaller than " + largeInt);
    }

    if (smallInt > largeInt) {
        System.out.println("This will never execute");
    } else {
        System.out.println(smallInt + " is smaller than " + largeInt);
    }
  }
}

3.10.2. Looping Flow Control

Looping flow control is performed by the for, do-while or while commands. The do-while flow control will guarantee execution of the block of statements at least once. All looping flow controls will repeat execution of the block of statements until the boolean value is false.

Figure 3.15. Looping Flow Control

figures/fundamentals/LoopingFlowControl.java
          
public class LoopingFlowControl {

  public static void main(String[] args) {

    int i = 0;

    // for loop example
    System.out.println("\nFor");

    for(int j = 0; j < 10; j++) {
      System.out.print(j);
    }

    // while example
    i = 0;
    System.out.println("\nWhile");

    while(i < 10) {
      System.out.print(i++);
    }

    // do while example
    i = 0;
    System.out.println("\nDo While");

    do {

      System.out.print(i++);
    } while(i < 10);
  }
}

The switch flow control will execute a block of code depending on the value of an attribute.

public class SwitchFlowControl {

    for(int i = 0; i < 6; i++) {

          System.out.println("Case 1: " + i);
        break;

        case 2:

          System.out.println("Case 2 or 3: " + i);

3.10.4. Break and Continue

There are times when you with to leave a block of code before the logical decision. Java has a break command which when encountered will automatically end processing of the block (break out of the block) and execute the next statement outside of the block. The break command can also break to a label in the program.

Java also has a continue command that stops the current iteration and starts with a new iteration. The continue command can also continue to a label in the program where the code is executed.

Break breaks out of the block. Continue stops executing in the block, increments or decrements the attribute and starts at the beginning of the block. Continue doesn't break out of the block.

Figure 3.17. Break Continue Label Flow Control

figures/fundamentals/BreakContinueLabelFlowControl.java
          
public class BreakContinueLabelFlowControl {

  public static void main(String[] args) {

    int   year     = 1;
    int   goal     = 100000;
    float balance  = 0f;
    float interest = 0.0725f;
    float payment  = 2000f;

    while (year <= 30) {

      balance = (balance + payment) * (1 + interest);

      // if loan paid off then break out of loop
      if (balance > goal) break;

      // if 30 years of payments then continue out of loop
      if (year == 30) continue;

      System.out.println(
        "year: " + year + 
        "\tbalance: " + balance + 
        "\tpayment: " + payment);

      year++;
    }
  }
}

3.11. Arrays

Arrays are a number of similar items arranged in a specific order. Arrays are navigated by index keys, starting with position 0 and incremented by integer values. Mentioned before C/C++ doesn't have a String class so C/C++ use an array of characters to store values as a String.

Figure 3.18. Arrays

figures/fundamentals/Arrays.java
          
public class Arrays {

  public static void main(String[] args) {

    int[]   arrayOfIntegers = new int[10];
    float[] arrayOfFloats = new float[10];

    for (int i = 0; i < 10; i++) {
      arrayOfIntegers[i] = i;
      arrayOfFloats[i] = i * 13.64f;
    }

    for (int i = 0; i < 10; i++) {
      System.out.println(arrayOfIntegers[i] + " " + 
        arrayOfFloats[i]);
    }
  }
}
