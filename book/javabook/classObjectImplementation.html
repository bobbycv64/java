4.1. Review

Recall Chapter 1 our discussion of Classes, Objects, Methods and Attributes. The Class is a framework that encapsulates attributes and methods. The Class defines how an object is to be managed on a system, e.g. Class Vehicle. A class is a blueprint for defining the methods and attributes for a particular type of object. All objects of a given class are identical in form and behavior but contain different data in their attributes. The class defines all common properties of different objects that are instances of the class.

Figure 4.1. new keyword

Objects are an instance of a class which encapsulate state and behavior using Information Hiding, e.g. Object Geo Metro is an instance of Vehicle. The example illustrates that all data attributes are hidden from all outside programs.

Objects are units of code that are eventually derived from the process. An object is a software packet containing a collection of related data in the form of attributes and methods for operating on that data. Objects are created with Constructors.

Figure 4.2. Object Anatomy

Methods are the behavior that occur within a class. Methods can receive zero to many arguments as input and return zero to one result as output. They encapsulate and hide the internals of the operation from the caller. There are accessor methods which get information and usually start with get or is (boolean return value) and only return information. Mutator methods set information and start with set, passing in information and returning no value.

A method is programmed logic that is defined as part of a class and included in any object of that class. Communication between objects take place between methods.

Figure 4.3. Method Types

Attributes are the state of data, e.g. the light bulb is either on or off. With OOP the attributes are hidden while SP left attributes vulnerable to being changed from outside references. Attributes can be built-in data types, such as a number or date, or reference another object.

4.2. new Keyword

Recall Chapter 3 on data types. Objects are types and are defined by the class they represent. For example the following statement Book book; States that the object book is an instance of Book the class. Book the class is a type and book the object is a type of Book.

Now that you understand what Classes and Objects are how do you create and use them. The answer is just like initializing primitive types in chapter 3. You must declare the object and give the object definition.

To declare an object just put the class type in front of the object name, e.g. Book book; This defines that the object book will be of type Book. The object book is only defined but not created.

The process of creating an object is called construction. To construct objects you either have to define your own constructors or the default constructor will create the object for you. To construct an object you must use the keyword, new.

public class New {

    int int1;					// declare the primative integer type

    int1 = 10;					// assign 10 to the primative integer i

    int int2 = 10;				// declare and assign to the primative integer j

    String string2 = new String("message");// declare and assign to the object string



    String string3 = null;			// declare and point to no object



    System.out.println("Integer1: " + int1);

    System.out.println("Integer2: " + int2);

    System.out.println("String2:  " + string2);

    System.out.println("String3:  " + string3);

  }

}

The Java language uses a similar concept as the C++ language. You can always create objects and allocate memory with Java and never be concerned about memory management. However with C++ you must also execute the keyword (delete) to release memory. Recall that the Java garbage collector automatically performs this task for you.

Looking at the Library class some items are new. Let's go line by line and figure out how to develop our own classes. The first line is the class declaration. The statement, "public class Library", states that a class named Library which must be in a file call Library.java is publicly accessible to all programs.

The next section is the Constructor stated as public Library() {}. Constructors create objects, instances of the class. When creating an object you must perform the operation through a constructor. In the C++ language you must also use destructors but with Java the Garbage Collector does this for you.

Java does have an option to alert the JVM you wish an object to be garbage collected using the finalize() method. This doesn't mean that the statements in the method occur immediately, only to alert the garbage collector to release memory.

The most basic constructor you can create contains no input attributes or implementation. The constructor statement is saying, "The Library class is accessible to any program, takes no arguments hence () has no implementation {} and constructors never return values. Constructors are a type of method with no return value. Remember the most important thing to remember is constructors construct objects, that is their sole purpose. We will learn more about constructors and complexities later.

The next section are the accessor methods, methods that return information. Accesor methods begin with get or is (boolean). This is a common programming standard but not required with the exception of Javabeans. All accessor methods return values and are public.

The first accessor getBook() states, "This method is publicly available, returns a String type and the value that is returned is the variable book_. A note. The class variables (this variables) all end with the underscore character (_, \u0000). I recommend others to use the underscore pattern instead of the this.variable. We will learn about "this" later because the discussion will require a section alone.

The next section are the mutator methods, methods that receive information. All the mutator methods begin with set. This is a common programming standard but not required with the exception of Javabeans. All mutator methods return nothing (void) and are public.

The first mutator method setBook(), states, "This method is publicly available, returns nothing (void) accepts a String and will declare the string as book. Notice that between the parenthesis ({}, \u0000\u0000) this is a block implementation. The method's implementation is saying take the value that was passed into the method and assign to the classes variable book_.

The last section are the attributes. The Library class has three attributes: book_, category_ and checkedOut_. These are internal to the class and only to the class because they all begin with the private reserved word. Recall chapter 1 with information hiding. All of the attributes for the Library class are hidden from outside interference because they are all private.

Finally, all the constructors, methods and attributes are encapsulated within the Library class. Before continuing you must understand the basic Object-Oriented concepts from Chapter 1 and what was just discussed.

public class Book {

  

  // 4 argument Constructor

  public Book(String author, String name, String isbn, float price) {

    author_ = author;

    name_ = name;

    isbn_ = isbn;

    price_ = price;

  }



  // 2 argument Constructor

  public Book(String author, String book) {

    this(author, book, "", 0.0f);

  }



  // no argument Constructor

  public Book() {

    this("", "");

  }



  // Accessor Methods

  public String getAuthor() { return author_; }

  public String getName()   { return name_; }

  public String getISBN()   { return isbn_; }

  public float  getPrice()  { return price_; }

  public Chapter getChapter(int chapter) {

    return chapter_[--chapter];

  }



  public String toString() {

    return(author_ + "\n" + name_ + "\n" + isbn_ + "\n" + price_);

  }



  // Mutator Methods

  public void setAuthor(String author) { author_ = author; }

  public void setName(String name)     { name_ = name; }

  public void setISBN(String isbn)     { isbn_ = isbn; }

  public void setPrice(int price)      { price_ = price; }



  public void addChapter(Chapter chapter) { chapter_[i_++] = chapter; }



  // Attributes

  private Chapter[] chapter_ = new Chapter[20]; // max chapters is 20

  private int i_ = 0;            // set ints to 0

  private String author_ = "";   // set strings to blank

  private String name_   = "";   // set strings to blank

  private String isbn_   = "";   // set strings to blank

  private float  price_  = 0.0f; // set floats to 0.0

}

4.4. What's this all about?

Java and C++ both have the concept and reference called, this. This means simply this object, the object itself. The this reference is implicitly known when an instance of an object is created and is used both for methods and constructors.

To simplify what this is all about I use a concept of the underscore at the end of a variable. The variable with the underscore is also this, for example: book_ could also be written as this.book_.

Many books written on the Java language will have the methods look like the first illustration below, using the this keyword or not at all, both are legal. I however like the ClearerThis example because you explicitly know which variable you are working with, a passive variable or a this variable, the variable with the underscore (_, \u00).

4.5. Accessibility

Java has five different approaches to accessibility. We will only mention two at this time. The public keyword for classes, methods and attributes means that any object can access that item. The private keyword for classes, methods and attributes means that only the class that defines each of these items can access that particular item. The following code illustrates public and private accessibility.

Figure 4.13. Constructor This

figures/classObject/Accessibility.java
          

public class Accessibility {



  // Anyone can access these items  

  public Accessibility() {}

  public void anyoneCanUseMe() {}

  public int anyoneCanUseMe_ = 0;



  // Only the object of this type can access these items

  private Accessibility(int someInteger) {}

  private void onlyAccessibilityCanUseMe() {}

  private int onlyAccessibilityCanUseMe_ = 0;

}

4.6. static Keyword

The static keyword is simply a class attribute or method and can never be an object (instance) attribute or method. The following example shows how you could use the static attribute to keep count of how many instances were created. Every time the class gets instantiated, another words new Static() the numberOfInstances_ attribute will increment. The instantiated objects then can get the total number of instances.

Figure 4.14. Static Keyword

figures/classObject/Static.java
          

public class Static {



  public Static() {

    numberOfInstances_++;

  }



  public static int getNumberOfInstances() {

    return numberOfInstances_;

  }



  private static int numberOfInstances_ = 0;

}
