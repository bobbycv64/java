6.1. Interfaces

Recall from Chapter 1, an interface defines a protocol of behavior that can be implemented by any class anywhere in the class hierarchy. An interface defines a set of methods with no implementation. A class that implements the interface agrees to implement all the methods defined in the interface, thereby agreeing to certain behavior.

Interfaces are a variation of Abstraction. All methods in an interface have no implementation, another words only the signature of the method is defined. Interfaces can have attributes but they must be a constant value.

Java allows only single inheritance, however, Interfaces are used to bypass this concept. A class can only extend one class and zero to many interfaces.

6.3. Inner Classes

Inner classes are a class or classes that are implemented within a class. When an object is created in a class an inner class has accessibility to this information as if it were its own object. Inner classes help with the implementation of callback methods that we saw earlier in interfaces. Inner classes can be hidden from other classes within a package. Inner classes take advantage of the Java 1.1 event model.

When we say hidden inner classes can hide the functionality completly from outside the class. The inner class can have the private accessor defined which means only the wrapper class can access the information. All classes with the exception of inner classes must allow some access or they are of no use.

Inner classes also can implement the singleton pattern. Look at the private inner class and notice that all the internal variables are public not private. This is due to the external classes not the outer class have no access to the private class.

Figure 6.6. JavaBean Class

figures/interfaces/JavaBean.java
          

// Interface Example

public interface JavaBean {



  public String get();

  public void set(String value);

}

Figure 6.7. Outer Class Example

figures/interfaces/OuterClass.java
          

// Inner Class Example

public class OuterClass {



  public JavaBean getInnerClass() {

    return new InnerClass();

  }



  private class InnerClass implements JavaBean {

    public String get() { return value_; }

    public void set(String value) { value_ = value; }

  }



  private String value_;

}

Figure 6.8. Inner Class Example

figures/interfaces/InnerClassExample.java
          

// Inner Class Example

public class InnerClassExample {



  public static void main(String[] args) {



    OuterClass outerClass = new OuterClass();

    JavaBean javaBean = outerClass.getInnerClass();

    javaBean.set("Inner Class Example");

    System.out.println("Inner Class Value: " + javaBean.get());

  }

}
